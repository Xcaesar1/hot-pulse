我作为一个 AI 编程相关从业者，要第一时间获取一些热点（比如 AI 大模型的更新），我不能依赖于人工手动搜索，而是利用工具自动发现指定的热点变化，或者发现最新的热点，并且及时给我发送通知，让我能够走在吃瓜第一线。

💡 由于是一个比较轻量的小需求（工具类项目），所以打算采用较为敏捷的开发方式，不用过多地人工介入和工程化。

## 具体来说，几个功能点：

- 用户手动输入要监控的关键词，当这个关键词相关的内容真的出现（注意利用 AI 识别假的内容），第一时间发送通知
- 每隔一段时间，自动搜集用户输入的指定范围（比如“AI 编程”）内的热点，并且能够让用户看到

## 产品的形式：

- 响应式兼容的 Web 页面
- 封装成 Agent Skills 技能，能够交给其他的 AI 来监控和发现热点

# 需求分析的技巧

对于工具类项目，需求分析的核心是抓住 **用户痛点** 和 **使用场景** 。鱼皮的痛点很明确：手动搜索热点效率低、容易遗漏、无法 7×24 小时监控。

在做需求分析时，建议思考以下几个维度：

| 维度       | 问题             | 本项目的回答                                          |
| ---------- | ---------------- | ----------------------------------------------------- |
| 目标用户   | 谁会用这个工具？ | AI 编程博主、科技媒体从业者、投资人等需要追踪热点的人 |
| 核心价值   | 解决什么问题？   | 自动化热点发现，减少人工搜索时间                      |
| 产品形态   | 用什么方式呈现？ | Web 页面 + Agent Skills 技能包                        |
| 技术可行性 | 能否实现？       | 利用公开 API + 爬虫 + AI 分析，完全可行               |
| 优先级     | 先做什么？       | 先做 Web 端核心功能，再封装 Skills                    |
