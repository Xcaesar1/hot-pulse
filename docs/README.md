## 建议阅读顺序
1. 先读需求文档，确认产品边界、目标用户和非功能要求。
2. 再读实施方案，理解整体架构、阶段划分和交付策略。
3. 接着看技术决策记录，确认哪些实现方式是基于 MCP / 官方文档校准后的最新做法。
4. 最后看实现状态和前端刷新记录，快速了解当前 Web MVP 已经做到哪里、这轮视觉优化改了什么。
