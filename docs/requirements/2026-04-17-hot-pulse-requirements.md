# Hot Pulse 需求文档

## 背景
面向 AI 编程从业者、科技媒体从业者、投资人等需要第一时间获取热点的人群，提供一个自动化的热点发现与通知工具，减少人工搜索成本，降低遗漏风险，并支持持续监控。

## 产品目标
- 自动发现指定关键词或主题范围内的新热点。
- 利用 AI 对候选内容进行相关性、真假风险、摘要判断。
- 从多个信息源交叉验证，降低单一来源误判。
- 在 Web 端完成配置、查看、通知和人工验收。
- 在 Web 能稳定使用后，再把同一套能力封装为 Agent Skill。

## 第一阶段范围
第一阶段仅交付单用户、本地优先运行的响应式 Web 应用，不包含注册登录和 Agent Skill。

## 核心功能
### 1. 监控规则
- 支持关键词监控，如 `Claude Code`、`GPT-5.4`。
- 支持主题监控，如 `AI 编程`、`开源 Agent`。
- 每条规则支持设置别名、通知开关、评分阈值。
- 默认热点检查频率固定为 `30 分钟`。

### 2. 多源采集
- 网页搜索发现链路：
  - DuckDuckGo HTML / Lite 搜索页
  - Google News RSS 查询页
  - 搜索结果落地页正文抓取
- X / Twitter 数据链路：
  - 通过 `twitterapi.io` 获取高级搜索结果和重点账号最近推文
- 结构化信息源：
  - Hacker News
  - Reddit
  - GitHub Releases
  - 用户手工录入的 RSS / 官方博客 URL

### 3. AI 判定
- 基于 OpenRouter 调用模型，对候选内容输出结构化分析：
  - `isRelevant`
  - `relevanceScore`
  - `credibilityRisk`
  - `summary`
  - `noveltyScore`
  - `suggestedNotify`
- 结合来源多样性、来源权重、传播速度计算最终热点等级。

### 4. 通知
- 默认提供站内通知与邮件通知。
- 通知分为 `High`、`Medium`、`Low` 三档。
- 支持发送测试通知。

### 5. Web 页面
- 实时热点面板
- 监控规则管理
- 来源配置与启停
- 命中历史 / 通知历史
- 任务运行状态

## 非功能要求
- 界面必须有辨识度，不使用通用后台模板。
- 本地优先，但架构上支持后续迁移到 Docker/VPS 常驻运行。
- 采集器要有频率限制、超时、失败回退和来源白名单。
- 所有外部集成方式优先参考 MCP 获取的最新实现方法，再结合官方文档校准细节。

## 成功标准
- 用户可创建监控规则并看到热点结果。
- 用户可手动触发扫描，并可查看来源证据和 AI 摘要。
- 用户可启停来源并看到最近一次运行状态。
- 多源交叉验证生效，单一低可信来源不会直接触发高优先级通知。
- 邮件配置存在时可收到测试通知与真实命中通知。
